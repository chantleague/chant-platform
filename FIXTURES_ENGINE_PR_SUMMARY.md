# Fixtures → Battles Automation Engine (Option A)

## Overview

This PR implements a comprehensive fixtures-to-battles automation system that:
- Fetches fixtures from multiple sources (BBC & Sky RSS adapters as stubs)
- Reconciles fixtures using intelligent deduplication
- Automatically creates battles for upcoming fixtures
- Supports manual admin overrides
- Provides a club-specific user experience

## Key Features Implemented

### 1. **Canonical Club Registry** (`app/lib/canonicalClubRegistry.ts`)
- Centralized registry for all clubs (PL, Championship, International, European)
- Supports stable, immutable slugs separate from display names
- Future-proof architecture for promotions/relegations without breaking URLs
- Helper functions: `getClubBySlug()`, `getClubsByLeague()`, `isValidClubSlug()`

**Example:**
```typescript
const arsenal = getClubBySlug("arsenal");
// Returns: { slug: "arsenal", displayName: "Arsenal FC", league: "PL" }
```

### 2. **Multi-Source Fixture Adapters** (`app/lib/fixtureSourceAdapters.ts`)
- `FixtureSourceAdapter` interface for extensible fixture sources
- Two stub implementations: `BBCRssAdapter`, `SkyRssAdapter`
- Return sample fixtures matching canonical club slugs
- Structure ready for real RSS/API parsing

**Extensible pattern:**
```typescript
interface FixtureSourceAdapter {
  fetchFixtures(): Promise<Fixture[]>;
  readonly name: string;
}
```

### 3. **Intelligent Fixture Reconciliation** (`app/lib/fixtureReconciliation.ts`)
- Fetches from all adapters in parallel
- Deduplicates by `(homeSlug, awaySlug, kickoffTime)` with configurable tolerance (default: 5 min)
- Filters out past fixtures and those beyond `maxDaysAhead`
- Filters fixtures to keep most-recently-fetched data
- Returns sorted, unified fixture list

**Reconciliation in action:**
```
BBC: arsenal vs brighton @ 15:00:00
SKY: arsenal vs brighton @ 15:04:30
↓ (time tolerance 5 min)
→ MERGED: arsenal vs brighton @ 15:00:00 (BBC source)
```

### 4. **Battle Creation Scheduler** (`app/lib/battleScheduler.ts`)
- Creates battles for fixtures within N days (default: 3 days)
- **Manual override support**: Battles marked `manuallyCreated: true` are never overwritten
- **Idempotent**: Running repeatedly doesn't duplicate battles
- Generates deterministic slugs: `{home}-vs-{away}`
- Metadata tracking: `fixtureId`, `autoGenerated`, `createdAt`, `updatedAt`

**Example workflow:**
```
1st run: Creates battle "arsenal-vs-brighton"
2nd run: Detects existing, skips if manually created, updates if auto-generated
3rd run: Only updates, zero duplicates
```

### 5. **Admin API Endpoint** (`app/api/admin/run-fixtures-sync/route.ts`)
- `POST /api/admin/run-fixtures-sync`
- Requires `Authorization: Bearer admin-token-stub` (stub auth for now)
- Orchestrates: reconciliation → battle creation
- Returns summary: `{ fixturesReconciled, battlesCreated, battlesUpdated, battlesSkipped }`

**Example response:**
```json
{
  "success": true,
  "message": "Fixtures sync completed",
  "summary": {
    "fixturesReconciled": 5,
    "battlesCreated": 3,
    "battlesUpdated": 0,
    "battlesSkipped": 2
  }
}
```

## Architecture & Design Decisions

### **Option A Selected** ✓
- ✓ Multi-source reconciliation (BBC + Sky RSS stubs)
- ✓ Stable canonical registry for future growth
- ✓ Manual admin override support
- ✓ Idempotent automation
- ✓ Extensible adapter pattern

### **Why These Patterns?**

1. **Canonical Registry**: Enables URL stability if clubs rename or move leagues
2. **Adapter Pattern**: Allows adding ESPN, Flashscore, etc. without core changes
3. **Reconciliation with Tolerance**: Real-world feeds have timing variance
4. **Manual Override Metadata**: Admin edits aren't destroyed by automation
5. **Idempotency**: Safe to run on a schedule (cron, webhooks, button press)

## Notes for Reviewers

1. **Stub Adapters**: BBC and Sky adapters are intentionally stubbed. They return sample data and log their execution. Real RSS parsing can be added later.

2. **Idempotency Test**: The smoke tests demonstrate idempotency by calling the admin endpoint twice and verifying the second call doesn't create duplicates.

3. **Club Registry Expansion**: The registry is designed to grow. Adding new leagues (League Two, Ligue 1, etc.) requires only adding a new list and including it in CANONICAL_CLUB_REGISTRY.

4. **Time Tolerance**: Set to 5 minutes (300 seconds). Can be tuned based on real-world data.

5. **Admin Auth Stub**: Currently checks for `Authorization: Bearer admin-token-stub`. Production should use proper JWT.
